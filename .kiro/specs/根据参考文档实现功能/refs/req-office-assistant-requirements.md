# 办公智能助手 Web 演示系统需求文档

## 项目概述

**项目名称**: Office Assistant (办公助手)  
**项目类型**: Web 前端演示系统  
**技术栈**: React + Ant Design  
**后端策略**: Mock 数据

## 目标

开发一个办公智能助手 Web 演示系统，用于展示核心办公功能，后端接口全部使用 Mock 数据。

## 功能模块

### 1. 首页仪表盘 (Dashboard)

- 今日待办事项统计
- 待审批数量提醒
- 日程概览（今日/本周）
- 快捷入口（常用功能）
- 公告通知轮播

### 2. 日程管理 (Schedule)

- 日历视图（月/周/日）
- 新建/编辑/删除日程
- 日程提醒设置
- 会议预约
- 日程分类（工作、会议、个人）

### 3. 审批流程 (Approval)

- 我发起的审批
- 待我审批
- 已审批记录
- 审批类型：请假、报销、采购
- 审批详情查看
- 同意/拒绝操作

### 4. 通讯录 (Contacts)

- 组织架构树形展示
- 员工列表（部门筛选）
- 员工详情（姓名、职位、电话、邮箱）
- 搜索功能
- 快速拨打/发邮件

### 5. 车辆管理 (Vehicle)

- 车辆列表（车牌、车型、状态）
- 车辆状态：可用、使用中、维修中、充电中
- 车辆详情（里程、电量、位置）
- 车辆筛选（按状态、车型）

### 6. AI 智能问答 (AI Assistant)

- 对话界面
- 常见问题快捷入口
- 支持文字输入
- 模拟 AI 回复（Mock）
- 历史对话记录

## 页面布局

- 顶部导航栏：Logo、搜索、通知、用户头像
- 左侧菜单：六个功能模块入口
- 主内容区：各模块页面内容

## 技术要求

- React 18+
- Ant Design 5.x
- React Router 路由管理
- Mock.js 或静态 JSON 模拟数据
- 响应式布局（适配 PC 端）

## 交付物

- 完整前端代码
- Mock 数据文件
- 项目运行说明
